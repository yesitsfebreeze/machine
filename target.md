we want this repo to behave like a real senior programmer we give jobs to.

- he writes concepts
- he writes implementations plans
- he implements those
- he tests them
- he lets his personas analyze it
- he evaluates what needs to be changed
- he implements the adjustments/fixes
- he presents it and closes it

the idea is that all of this can happen in a subagent that we can communciate with.
so we dispatch multiple tasks and implement them using git-fs
we keep track of open features, and let agents communicate between each other so we dont do duplicate implementations
