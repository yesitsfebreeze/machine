# Hub Mine System â€” Implementation Plan

## 1. Mine Catalog Scanner (hub/src/mine.rs)

Introduce `hub/src/mine.rs`. This module owns all catalog logic: scanning, parsing,
and computing install status.

PLUGIN_ROOT resolution via three-step cascade: first check `CLAUDE_PLUGIN_ROOT` env
var; second check `--plugin-root` CLI flag parsed from the args slice; third derive
by walking up from `std::env::current_exe()` looking for a directory containing
`mine/`, capping at three levels. If all three fail, `scan_catalog()` returns an
empty Vec and logs a warning to stderr â€” not a fatal error.

`scan_catalog(plugin_root: &Path, project_cwd: &Path) -> Vec<MineItem>` iterates two
subdirectories: `<plugin_root>/mine/agents/` and `<plugin_root>/mine/skills/`. For
agents, each `.md` file at the top level is one catalog item (name = stem). For
skills, each subdirectory containing a `SKILL.md` is one catalog item (name = dir
name).

Frontmatter parsing: open the primary file (agent `.md` or skill `SKILL.md`), read
line by line until the closing `---` delimiter, extract `name:` and `description:`
keys. No full YAML library required â€” line-by-line scan collapsing multi-line scalars.
Malformed or missing files are skipped with a stderr warning; scan continues.

Install status: for each item, `installed` is computed fresh on every call by checking
whether `<project_cwd>/.claude/agents/<name>.md` exists (agents) or
`<project_cwd>/.claude/skills/<name>/` exists (skills). Simple `Path::exists()`, no
caching.

## 2. MineItem Data Model

Defined in `hub/src/mine.rs`. Enum `MineType` with variants `Skill` and `Agent`.
Struct `MineItem` carries: `name: String`, `item_type: MineType`,
`description: String`, `source_path: PathBuf` (absolute path in mine, internal only),
`installed: bool`. Method `to_json(&self) -> Value` serializes to
`{ "name", "type" ("skill"/"agent"), "description", "installed" }`. `source_path` is
internal and does not appear in JSON responses.

---

## 3. install.toml Schema and I/O

File at `<project_cwd>/.machine/install.toml`. Two TOML tables: `[skills]` with
key `installed` (array of strings) and `[agents]` with key `installed` (array of
strings). Missing file = empty manifest (normal for new projects). Malformed file =
`Err` with path and parse error surfaced as a tool error.

`InstallManifest` struct holds `skills_installed: Vec<String>` and
`agents_installed: Vec<String>`. Uses `toml` crate (add to `hub/Cargo.toml`).

`InstallManifest::load(project_cwd: &Path) -> Result<InstallManifest>`: absent file
returns empty manifest, malformed returns Err.

`InstallManifest::append(project_cwd: &Path, item_type: MineType, name: &str) -> Result<()>`:
load current (or start empty), check if name already present, push if not, serialize
back to disk. Parent `.machine/` created with `fs::create_dir_all` if absent. Write
is atomic: temp file + `fs::rename`.

## 4. hub_mine_list Tool

Added to `preload_verbs()` in `registry.rs`. Input schema: empty object (no required
fields).

Handler calls `mine::scan_catalog(&plugin_root, &project_cwd)`. Response: JSON object
with single key `"items"` whose value is an array. Each element carries `name`, `type`
(`"skill"` or `"agent"`), `description`, `installed` (boolean).

Sort order: agents before skills; within each type group, sorted lexicographically by
name.

---

## 5. hub_mine_install Tool

Added to `preload_verbs()`. Input schema: object with two required string properties:
`type` (enum `"skill"` or `"agent"`) and `name`.

Validation: scan catalog to verify item exists. If not found, return tool error. If
already installed, return `{ "status": "already_installed", "name": ..., "type": ... }`
(success, not error).

Copy behavior: for a skill, recursively copy `<plugin_root>/mine/skills/<name>/` to
`<project_cwd>/.claude/skills/<name>/`. For an agent, copy
`<plugin_root>/mine/agents/<name>.md` to `<project_cwd>/.claude/agents/<name>.md`.
Destination parent created with `fs::create_dir_all`. Recursive dir copy implemented
with `std::fs::read_dir` walker â€” no external crate.

install.toml update: call `InstallManifest::append` after successful copy. If append
fails, log to stderr but do not reverse the copy â€” files are source of truth.

Registry update: not performed automatically on install (noted as future extension
point in a comment).

Response on success: `{ "status": "installed", "name": ..., "type": ... }`.

## 6. hub_mine_restore Behavior

Added to `preload_verbs()`. Input schema: empty object.

`mine_restore(plugin_root: &Path, project_cwd: &Path) -> Result<Value>`: load
`InstallManifest::load(project_cwd)`. For each name in `agents_installed`: if
`<project_cwd>/.claude/agents/<name>.md` does not exist, locate source at
`<plugin_root>/mine/agents/<name>.md` and copy. If source missing from mine, log
warning and skip (no error). For each name in `skills_installed`: if
`<project_cwd>/.claude/skills/<name>/` does not exist, recursively copy from mine.
Count restored and skipped separately.

Idempotency: the `!path.exists()` check before each copy guarantees already-present
items are never overwritten.

Response: `{ "restored": N, "skipped": M }`.

Startup hook: in `board_http::serve_http`, after opening Board state and before
binding the TCP listener, call `mine_restore` if `.machine/install.toml` exists at
cwd. Result is logged to stderr. A restore failure logs to stderr but does not abort
startup.

## 8. Mine Panel in Board UI

`hub/src/board_ui.html` gains a tab bar above the existing board content with two
entries: "Board" (active by default) and "Mine". Tab switching toggles a CSS class on
a container div.

Mine tab: on activation, fires `GET /api/mine/list` (new REST route in
`board_http.rs`). Items rendered as cards in a two-column grid. Each card shows: item
name (heading), type badge (pill span, different color for skill vs agent), description
text. Installed items show a muted "installed" badge. Uninstalled items show an
"Install" button.

Install flow: clicking Install fires `POST /api/mine/install` with JSON body
`{ "type": ..., "name": ... }`. On `200` response, re-fetch `GET /api/mine/list` and
re-render. If install POST returns an error body, show inline error below the clicked
card. No WebSocket push for mine installs â€” simple re-fetch is sufficient.

New routes added to `board_http.rs` router:
- `GET /api/mine/list` â†’ calls `hub_mine_list` handler
- `POST /api/mine/install` â†’ calls `hub_mine_install` handler

## 9. hub.toml Schema Definition

File resolution order: `<plugin_root>/hub.toml` first, then `<project_cwd>/hub.toml`,
then `<project_cwd>/.machine/hub.toml`. First found wins. None found = empty proxy
list.

Top-level key is `[mcps]`. Each entry is `[mcps.<name>]`. Stdio form: `command`
(string, required), `args` (array of strings, optional, default empty). HTTP form:
`url` (string, required). Both forms support optional `env` (inline TOML table,
stringâ†’string). Mixing `command` and `url` in the same block is a parse-time error.

Load at startup in `board_http::serve_http` after the restore step. Stored in
`HubState`. Malformed file logs parse error and proceeds with empty proxy list. Used
only by proxy layer when implemented (out of scope for this plan).

## 10. Test Approach

Unit tests in `hub/src/mine.rs` under `#[cfg(test)]`. Fixture-based via `tempdir`.
Synthetic mine: two agent `.md` files with YAML frontmatter, two skill dirs each with
`SKILL.md`. Tests: `scan_catalog` returns four items with correct names, types,
descriptions; marks installed when `.claude/` path exists; skips items with missing or
malformed primary file without panicking.

`InstallManifest` tests: round-trip (write, read back, assert match); idempotency
(append same name twice, appears once); missing file returns empty manifest;
corrupt TOML returns Err.

`hub_mine_install` tests: given fixture mine and project dirs, install copies skill
dir recursively and agent file; second call returns `already_installed` without
modifying files.

`hub_mine_restore` tests: write install.toml listing two items, ensure only one exists
in `.claude/`, call restore, assert missing one copied and present one untouched; call
restore again, assert `restored: 0, skipped: 0`.

Integration tests in `hub/src/server.rs`: call `hub_mine_list` via `handle_line` with
fixture PLUGIN_ROOT, assert correct items array; call `hub_mine_install`, assert copy
made and install.toml updated, call `hub_mine_list` again and assert installed flag
flipped to true.

Test isolation: all tests use explicit path arguments rather than env-var mutation.
Existing `COUNTER: AtomicU64` pattern reused for test port allocation.
