# Plan a1 ‚Äî Replace `trello` addon with local `taskboard` (MCP + web kanban)

Status: stored (advisory review pending). Source verified at /tmp/tb-probe.

## 1. Summary and verified taskboard facts

Swap the unregistered `trello` addon for a local, buildable **taskboard**
(`github.com/tcarac/taskboard`): one `taskboard` daemon serving all boards, the board
for the current repo selected by cwd (1 taskboard *project* = 1 cwd), mirroring the
drill's `.machine/sessions/` ledger. The `taskboard` MCP server is wired into
`plugin.json`, a new `mine/skills/taskboard/SKILL.md` documents the discipline,
bring-up provisions the binary + daemon, and per-cwd identity persists in
`.machine/taskboard.json` (replacing the never-present `.machine/trello.json`).

Verified (cited from /tmp/tb-probe):
- Go 1.24 / MIT. `go.mod:3`; `LICENSE`. Node 22 needed for the web frontend build
  (`Makefile:12-15`).
- One daemon serves many boards, board chosen by projectId. `internal/db/store.go:440-466`
  `GetBoard(projectID)`; `internal/mcp/mcp.go:288-293` `get_board` takes `projectId`.
- Three hard status columns, no custom columns. `internal/db/store.go:441`
  `statuses := []string{"todo","in_progress","done"}` (also mcp.go:470,496,513,528,545).
- MCP transport JSON-RPC over stdio. `internal/mcp/mcp.go:65-95`; `taskboard mcp` wires
  `mcp.NewServer(store).Run()` (`internal/cli/root.go:88-101`).
- Web daemon `taskboard start`, default :3010 (`root.go:33-50`); prints URL
  (`internal/server/server.go:36-40`). Daemonizes; `--foreground` runs inline.
- Both share one SQLite DB, `~/.config/taskboard/taskboard.db` (`internal/db/db.go:50-60`).
- Daemon idempotent / single-instance via pidfile `~/.config/taskboard/taskboard.pid`
  (`root.go:158-208`); refuses double-start.
- **projectId is a server-generated ULID, NOT derivable from cwd** (`store.go:88-90`,
  `47-49`); `prefix` is a separate free-text field (`store.go:104-106`). Must resolve
  by listing + matching, then persist.
- **`web/dist` is `//go:embed`** (`cmd/taskboard/main.go:9-10`): bare `go install` FAILS
  without the frontend build. Must `make build` (runs `make frontend` first). Node 22 is
  effectively a hard build dependency.

### 3.2 scripts/bootstrap.sh ‚Äî provisioning + idempotent daemon start
REVISED (prebuilt-first ‚Äî upstream ships release binaries; NO Go/Node needed).
Verified: v0.6.0 release has `taskboard-{darwin,linux}-{amd64,arm64}.tar.gz`; the
linux-amd64 binary is a 12M statically-linked ELF with the web frontend embedded and
all subcommands (`mcp`,`start`,`stop`,`project`,`ticket`). End-to-end proven live

Add `install_taskboard()`, called in the run block after `check_others`. Logic:
1. If `have taskboard` ‚Üí ok, skip to daemon.
2. **Primary ‚Äî download pinned prebuilt.** Map `uname -s`/`uname -m` ‚Üí
   {darwin|linux}-{amd64|arm64}; `curl -fsSL` the pinned-tag asset
   `https://github.com/tcarac/taskboard/releases/download/<PIN>/taskboard-<os>-<arch>.tar.gz`
   (PIN = `v0.6.0`, a pinned constant ‚Äî bump deliberately, never track latest), extract,
   install the binary RENAMED to `taskboard` into `$LOCAL_BIN`, `chmod +x`, verify on
   PATH. No Go, no Node. Verify the downloaded binary runs (`taskboard --help`).
3. **Fallback ‚Äî source build** only if the download fails AND go>=1.24 + node>=22+npm
   present (the `//go:embed web/dist` needs the frontend, so `make build`, not bare
   `go install`). If neither path works ‚Üí `warn "taskboard install skipped"` + return.
4. Daemon start ("one server if not up"): if pidfile holds a live pid ‚Üí ok; else
   `taskboard start` (:3010, idempotent ‚Äî refuses double-start) + bounded readiness poll.
   Treat "already running" as ok; port-bind failure ‚Üí warn.
5. Any failure ‚Üí `warn` (board projection disabled this session); never abort the whole
   bootstrap on taskboard alone (best-effort addon).
Update the header comment block to list taskboard.

### 3.5 .machine/taskboard.json ‚Äî schema + keying (replaces trello.json)
```json
{ "version":1, "cwd":"/abs/path", "name":"machine", "prefix":"P<hash>",
  "projectId":"<ULID>", "url":"http://localhost:3010", "resolvedAt":"<iso>" }
```
Keying: `name=basename(cwd)`; `prefix="P"+first 6 upper chars of a stable hash
(e.g. FNV/SHA-1) of absolute cwd` ‚Äî deterministic + collision-resistant. Look up by
prefix; verify persisted `cwd` matches before trusting projectId (collision guard);
create if absent; persist returned id.

## 5. Edge cases
Daemon already running (pidfile); binary missing + no Go (fail with URL, projection
off, skill degrades gracefully); Go present but Node missing (web embed can't build ‚Äî
warn, skip); project exists for cwd (match prefix, persist, skip create); :3010 taken
(warn; MCP card ops still work without UI); offline (warn, disable projection;
installed binary works fully offline); stale taskboard.json (re-resolve).

## 8. Codex review (advisory) ‚Äî folded notes

Codex was present; these amend the sections above. Material items folded:

- **8.1 Resolver owner (codex#1, strongest).** A SKILL is instructions executed by the
  drill agent (which calls `mcp__taskboard__*` + Write) ‚Äî that is the baseline writer,
  same as the retired trello skill. To remove ambiguity and give a deterministic,
  agent-independent path, ALSO add **`scripts/taskboard-resolve.sh`**: a thin helper
  that (given cwd) computes name+prefix+hash, lists/matches/creates the project via the
  taskboard CLI, and writes `.machine/taskboard.json`. The SKILL documents the same
  procedure and may call the helper. New file in ¬ß3 (call it ¬ß3.11).
- **8.2 Project identity in `description` (codex#3,#4).** At `create_project`, write the
  absolute cwd (or its full hash) into `description`. Resolve by: `list_projects` ‚Üí
  match on prefix AND verify `description` carries this cwd. If 0 matches ‚Üí create; if
  exactly 1 ‚Üí use; if >1 ‚Üí do NOT guess: require a persisted `projectId` or create with
  a more specific name. `.machine/taskboard.json.cwd` only guards the local file; the
  `description` is the canonical remote disambiguator.
- **8.3 Bootstrap is best-effort, consistently (codex#6,#7).** taskboard is an addon, not
  a core daemon: any install failure ‚Üí `warn` + return (NOT `fail`/abort). With the
  prebuilt-first strategy (¬ß3.2) the common path needs neither Go nor Node, so the
  Go/Node-missing case only matters for the source fallback.
- **8.4 Pin upstream (codex#8).** Do not track `main`/`latest`. Pin the release tag
  `v0.6.0` as a bootstrap constant for BOTH the prebuilt download asset and the source
  fallback checkout; bump deliberately. Reproducible.
- **8.5 Readiness check (codex#9).** After `taskboard start`, poll the pidfile + the web
  API for readiness (short bounded loop) before declaring the daemon up; daemonize can
  return before the server is listening.
- **8.6 Verify the real web API route (codex#12).** Do not assume `/api/board`. Read
  `internal/server/server.go` routes and use the actual board endpoint (and whether it
  needs a projectId) in the verification gate.
- **8.7 Key-list wording (codex#13).** In assemble.md, list taskboard under a
  "local services / no key required" row, NOT the required-key (secrets) list.
- **8.8 Narrow skill triggers (codex#16).** Description triggers: "taskboard",
  "taskboard board", "kanban board", "drill board" ‚Äî avoid bare "board"/"kanban" that
  could mis-activate.
- **8.9 Rollback note (codex#17).** Document rollback: remove the plugin.json taskboard
  entry, remove `mine/skills/taskboard/`, optionally `taskboard stop`; leave the SQLite
  DB untouched.
- **8.10 Grep variants (codex#14).** Verification grep must catch `trello`, `Trello`,
  `@mine/skills/trello`, `.machine/trello.json`; exclude `.git`, caches, build outputs,
  `.machine/worktrees/**`, `.claude/worktrees/**`.
- **8.11 SQLite concurrency (codex#11).** Note whether upstream uses WAL/busy-timeout for
  the shared DB; if unverified, expect possible lock contention on simultaneous UI+MCP
  writes ‚Äî implementer checks `internal/db/db.go`.
