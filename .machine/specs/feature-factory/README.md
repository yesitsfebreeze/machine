# Feature factory (SPEC-FACTORY-001)

The machine as a fleet of dispatchable senior programmers: one job = one
default-agent lifecycle (concept -> plan -> implement -> test -> persona analysis
-> evaluate -> fix -> present/close), isolated on a git-fs branch, deduped via mesh.

Document trail (the factory's own lifecycle, applied to itself):

| Stage | Doc |
|-------|-----|
| Concept | `CONCEPT.md` (CONCEPT-FACTORY-001) |
| Spec | `SPEC.md` (SPEC-FACTORY-001, EARS, R1-R14, AC1-8) |
| Plan | `PLAN.md` (PLAN-FACTORY-001, M1-M6) |
| Close | `CLOSE.md` (CLOSE-FACTORY-001, evidence + verdict) |

Operational behaviour lives in `.claude/agents/default.md` (the lifecycle + the
two dispatch roles + the handshake) and `.claude/skills/orchestrate/SKILL.md` (the
ledger schema + driver reconciliation). These docs are the design record; do not
duplicate operational rules here.
