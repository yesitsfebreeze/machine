# Project layer — the machine (read after the portable `default.md`)

## What this project is

This repo is **the machine itself** — the portable Claude Code agent setup. Unlike a
normal codebase, the "product" here is the agent configuration: Markdown instruction
documents, Node hooks, and `settings.json`. Editing this repo edits how the machine
behaves in every repo it is later installed into. Treat every instruction file as
production: a stray word changes agent behavior downstream.

The canonical truth is `.claude-plugin/plugin.json` (the plugin manifest),
`.claude/skills/oil-me/SKILL.md` (the re-index protocol), and
`.claude/agents/default.md` (machine law). Install/update is the `/plugin` system;
`/.machine/` is regenerated by `/oil-me`.

## Project law (binding — as hard as machine law)

These come from `.claude/rules/coding-standards.md` and the unification design:

1. **English-only instructions.** All instruction docs (CLAUDE.md, agents, commands,
   skills, hooks, config) must be English. User-facing docs (README/CHANGELOG) may differ.
2. **No external product naming.** Zero MoAI / MoAI-ADK / Spool / TRUST-5-as-branding /
   `<moai>` markers / non-existent `CLAUDE.md §N` references. Identity is "the machine";
   memory daemon is `kern`; project layer is `/.machine/`; rules are "Machine law".
3. **No emoji** in instruction documents (output styles excepted).
4. **Single source of truth.** Each fact lives in exactly one file; reference with
   `@file` instead of copying. Update the source, never a copy.
5. **Thin command pattern.** Slash-command files are routing wrappers < 20 LOC that
   invoke a skill via the Skill tool; all workflow logic lives in the skill body.
6. **File-size limits.** CLAUDE.md ≤ 40,000 chars; move overflow to `.claude/rules/`.
7. **Dispatch integrity is sacred.** Agents resolve by `name:` frontmatter, not path —
   every name must stay unique and every `skills:` reference must resolve to a real
   skill dir. Renames must update both sides in the same change.
8. **No content code-examples** for conceptual explanation; no flow-control or decision
   trees expressed as code; no time/duration estimates in instructions.

## Domain idioms

- **Markdown frontmatter is the API.** Agent/skill behavior is driven by YAML
  frontmatter (`name`, `description`, `allowed-tools`, `skills`, `paths`). Get the
  frontmatter right before the body.
- **Verify by parse, not by run.** "Done" means hooks pass `node --check`, `settings.json`
  parses, and agent/skill references resolve — there is nothing to execute.
- **Edit the machine surgically.** `/oil-me` re-index writes only under `/.machine/`; never let
  it touch `.claude/`. When improving the machine, change the one canonical file.
- **Commit per verified slice.** No CI exists; git history is the only safety net. The
  ongoing unification work is sliced in `.machine/improve.json`.
